# Бэкенд журнала работ (NestJS + TypeScript)

REST API сервер для учета выполненных работ на строительном объекте, спроектированный на NestJS, TypeScript, MySQL и Prisma ORM.

## Используемые технологии
*   `@nestjs/core`, `@nestjs/common` (основа веб-сервера)
*   `prisma` + `@prisma/client` (реляционное сопоставление и миграции)
*   `class-validator` + `class-transformer` (валидация входящих данных и DTO)
*   `jest` (юнит-тестирование)

## Запуск в режиме разработки
Для запуска бэкенда отдельно в режиме разработки:

1.  Установите зависимости:
    ```bash
    npm install
    ```
2.  Настройте подключение к MySQL в файле `.env` (параметр `DATABASE_URL`).
3.  Накатите схему БД и заполните справочник видов работ:
    ```bash
    npx prisma db push
    npx prisma db seed
    ```
4.  Запустите сервер с автообновлением при сохранении:
    ```bash
    npm run start:dev
    ```

Сервер запустится локально на порту `3001` (эндпоинты будут доступны по префиксу `/api`).

## Тестирование
Для запуска комплекта юнит-тестов выполните:

```bash
npm run test
```
