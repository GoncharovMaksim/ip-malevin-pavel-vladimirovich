# Журнал работ на строительном объекте

Fullstack-приложение для учета выполненных строительных работ на объекте.

## Технологический стек

### Фронтенд
* **React (TypeScript, Vite)**
* **Tailwind CSS**
* **Framer Motion** (анимации окон и списков)
* **TanStack Query** (управление серверным состоянием)
* **Lucide React** (иконки)

### Бэкенд
* **NestJS (TypeScript)** (модульная архитектура, DTO-валидация)
* **Prisma ORM**

### Инфраструктура
* **MySQL 8.0**
* **Nginx** (раздача статики и проксирование API на одном порту)
* **Docker & Docker Compose**

---

## Подготовка к запуску

Для запуска приложения необходимы:
1. **Docker**
2. **Docker Compose**

Все параметры конфигурации и порты заданы по умолчанию в корневом файле `.env`.

---

## Запуск приложения через Docker Compose

Для сборки и запуска всех контейнеров выполните в корне проекта:

```bash
docker compose up --build -d
```

При первом запуске:
1. Создаются и запускаются контейнеры для базы данных, бэкенда и фронтенда.
2. Применяются миграции базы данных.
3. Запускается seed-скрипт для наполнения справочника видов работ стандартными операциями (кладка, штукатурка, заливка бетона и т.д.).

После сборки приложение будет доступно по адресам:
* Фронтенд: **http://localhost**
* API бэкенда: **http://localhost:3001/api**

---

## Остановка и удаление

* **Остановить приложение**:
  ```bash
  docker compose down
  ```

* **Остановить и полностью удалить данные БД**:
  ```bash
  docker compose down -v
  ```

---

## Локальный запуск (для разработки)

### 1. База данных
Запустите локальный сервер MySQL, создайте базу данных `construction_log` и укажите строку подключения в `backend/.env`.

### 2. Бэкенд
```bash
cd backend
npm install
npx prisma db push
npx prisma db seed
npm run start:dev
```

### 3. Фронтенд
```bash
cd frontend
npm install
npm run dev
```
