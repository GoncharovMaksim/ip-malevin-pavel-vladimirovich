# Журнал работ на строительном объекте

Профессиональное fullstack-приложение для ведения учета выполненных строительных работ на объекте. Решение спроектировано с учетом всех современных практик веб-разработки, разделения зон ответственности и модульной архитектуры.

## Технологический стек

### Фронтенд
*   **React + TypeScript (Vite):** Быстрая сборка, строгая типизация интерфейсов, высокая отзывчивость.
*   **Tailwind CSS:** Современный адаптивный UI с лаконичным дизайном.
*   **Framer Motion:** Плавные микроанимации интерфейса (модальные окна, фильтрация, добавление и удаление записей).
*   **TanStack Query (React Query):** Оптимальное управление серверным состоянием, кэширование запросов, автоматическая инвалидация данных после мутаций.
*   **Lucide React:** Пакеты согласованных минималистичных векторных иконок.

### Бэкенд
*   **NestJS + TypeScript:** Модульный веб-фреймворк корпоративного уровня. Реализует паттерн Dependency Injection, DTO (Data Transfer Objects) и строгую валидацию входящих запросов на уровне контроллеров.
*   **Prisma ORM:** Удобное взаимодействие с реляционной базой данных, 100% типизация схем и запросов, поддержка миграций и сидов.

### База данных и Инфраструктура
*   **MySQL 8.0:** Надежная реляционная СУБД для хранения записей и справочника работ.
*   **Nginx:** Высокопроизводительный сервер для раздачи статических файлов фронтенда и проксирования запросов на API бэкенда (избавляет от необходимости настройки CORS в продакшене).
*   **Docker & Docker Compose:** Полная контейнеризация приложения для запуска одной командой на любом сервере.

## Подготовка к запуску

Для запуска приложения на вашем компьютере или сервере должны быть установлены:
1.  **Docker** (версии 20.10 или выше)
2.  **Docker Compose**

В корне проекта находится файл `.env`, содержащий конфигурацию портов и учетных данных базы данных. По умолчанию все параметры уже настроены для мгновенного запуска.

---

## Инструкция по запуску (Docker Compose)

Запуск всего приложения вместе с базой данных, бэкендом и фронтендом выполняется одной командой:

```bash
docker-compose up --build -d
```

При первом запуске:
1.  Поднимется контейнер базы данных MySQL.
2.  NestJS-сервер автоматически применит миграции (создаст таблицы `WorkType` и `WorkLog`).
3.  Запустится seed-скрипт, который наполнит справочник видов работ стандартными строительными операциями («Кладка перегородок», «Монтаж опалубки», «Штукатурка стен», «Заливка бетона» и т.д.).
4.  Скомпилируется фронтенд и запустится Nginx.

После завершения сборки приложение станет доступно в браузере по адресу:
👉 **[http://localhost](http://localhost)**

API бэкенда доступно напрямую по адресу:
👉 **[http://localhost:3001/api](http://localhost:3001/api)**

## Остановка и удаление приложения

*   **Остановить приложение** (данные в БД сохранятся):
    ```bash
    docker-compose down
    ```

*   **Остановить и полностью стереть все данные** (включая тома БД):
    ```bash
    docker-compose down -v
    ```

## Локальный запуск без Docker (для разработки)

Если вам необходимо запустить проект локально в режиме разработки (с поддержкой Hot Reload):

### 1. Подготовка базы данных
Убедитесь, что у вас запущена локальная СУБД MySQL. Создайте базу данных `construction_log` и настройте файл `backend/.env` с вашей строкой подключения `DATABASE_URL`.

### 2. Запуск Бэкенда
```bash
cd backend
npm install
npx prisma db push
npx prisma db seed
npm run start:dev
```
Сервер бэкенда запустится на порту `3001`.

### 3. Запуск Фронтенда
```bash
cd frontend
npm install
npm run dev
```
Фронтенд запустится в режиме разработчика (обычно на порту `5173`) и будет автоматически проксировать запросы к API на `http://localhost:3001/api`.
